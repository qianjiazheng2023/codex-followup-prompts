# Codex 追问建议

这个本地 Codex 插件会在每次最终回复的末尾生成三条“你可能接着问”。建议优先基于最近的对话内容，同时保持与整个任务目标一致。

## 实现方式

插件使用 `UserPromptSubmit` 生命周期钩子，在每轮开始前向 Codex 注入追问生成规则。模型本身已经拥有当前任务的对话上下文，因此不需要额外 API Key，也不会把聊天内容发送给第三方服务。

当前 Codex 的公开插件接口没有聊天气泡下方的按钮注入 API，因此建议以回复末尾的文本列表呈现；每一条都可以直接复制或发送。该限制只影响原生按钮外观，不影响按上下文生成追问。

## 安装与启用

1. 在 Codex 的插件页面安装 `codex-followup-prompts`。
2. 启用插件后，查看并信任它的 `UserPromptSubmit` 钩子。
3. 新开一个任务进行测试。新任务的每次最终回复都应含有“你可能接着问”及三条建议。

要临时关闭当前轮的建议，在消息中明确写“不要显示追问建议”即可。

## 验收示例

当回答刚解释完一个技术方案，建议应该类似：

- 把这个方案实现到当前项目
- 为关键流程补一组测试用例
- 比较两种实现的性能和维护成本

它们会随对话主题变化，不会固定输出以上内容。
